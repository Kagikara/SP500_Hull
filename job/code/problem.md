【A 级：致命错误（会导致全部结果失真）】
A1. 全样本标准化、分位数、排名 → 未来数据泄露（最严重）
错误描述

你对所有因子做分位数、rank、均值标准化、中心点计算时使用了整个数据集。

危险性

这等同于：

用未来10年的数据来定义当下因子的大小。

会让因子看起来比实际更能预测未来。

报告中出现方式

RankP、RankM 使用“全样本分位数”

标准化使用全样本均值/方差

阈值区间（0.52~0.575）来自全样本预测结果分布

正确做法

必须改成：

✔ 滚动窗口统计量（rolling mean/std/rank）
✔ 每日仅使用当天之前的历史数据计算所有因子

A2. 阈值扫描（threshold tuning）在测试集上进行 → 过拟合数据窥视
错误描述

你对 open_threshold 从 0.50~0.65 逐点扫描，并选择性能最优点。
这些扫描是在同一段数据（测试段）上完成的。

危险性

这是最典型的 data-snooping：

用测试集来调参 → 评价结果虚高、不可信。

报告中出现方式

0.5707、0.575 的最优阈值是反复试出来的

关闭阈值 close_threshold 也是扫描得到

正确做法

✔ 在训练集内（严格不包含未来数据）调参
✔ 测试集只用一次，不得回头调整

A3. 模型超参数网格搜索没有使用时间序列交叉验证（导致未来信息泄露）
错误描述

超参数（例如 RF 树数量、SVM 的 C/γ）是用固定训练集直接网格搜索出来的。

危险性

市场是时间序列，不能把未来数据当做独立同分布样本。

你现在的做法相当于“提前看到未来再决定参数”。

报告中的方式

对所有模型使用一个固定训练段（date_id >=1006）

在其内部做普通 K-fold 或无验证的 grid search

超参数固定后，再直接回测并评价

正确方法

✔ 使用 Purged K-fold (De Prado)
✔ 或 Walk-forward（滚动窗口）训练 + 验证

A4. 特征处理的顺序错误：先看全样本预测分布再设计策略
错误描述

你先：

训练模型

得到所有样本的预测概率

用全样本概率分布计算

中枢区间

分层收益（0.52 / 0.53 / 0.54）

开仓与加仓阈值

危险性

模型的未来预测被你拿来反推阈值，这相当于：

你提前知道模型在未来每一天的信心度，然后反向调策略。

这就是未来信息泄露。

正确方法

✔ 每个时间段单独训练
✔ 不得使用未来预测值来定义历史阈值

A5. 回测没有交易成本、滑点、换仓延迟 → 结果虚高
错误描述

回测假设：

全部在收盘价成交

0 成本

无滑点

杠杆动态调整无成本

无容量限制、无冲击成本

危险性

高换手策略（动态波动率缩放 + 阈值过滤）会受到巨大冲击成本。

没有成本会让策略表现远高于真实可实现值。

正确做法

✔ 加入固定交易成本（0.5~3 bps）
✔ 加入滑点
✔ 加入仓位跳变成本
